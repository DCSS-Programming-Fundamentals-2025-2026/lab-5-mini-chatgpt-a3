Lib.Batching (Етап 1 — Група А3)
Ця бібліотека відповідає за підготовку даних для тренування моделі у проєкті Mini-ChatGPT. Вона забезпечує формування тренувальних батчів — пар (context, target) — шляхом безпечного вибору випадкових вікон із загального потоку токенів.

Архітектура та Опис Класів
Проєкт розділений на логічні директорії та шари:

1. Domain & Application
IBatchProvider.cs та ITokenStream.cs: Основні абстракції системи, що визначають контракти для постачальників батчів та потоків токенів відповідно.

Batch.cs: Модель даних, що представляє сформований батч (містить масиви контексту та відповідних цільових токенів для передбачення).

TokenBatchProvider.cs: Головна реалізація IBatchProvider, яка об'єднує потік токенів та семплер для генерації готових батчів.

2. Configuration
Configuration/BatchConfig.cs: Клас для керування параметрами та налаштуваннями процесу батчингу.

3. Streams & Sampling
Streams/ArrayTokenStream.cs: Реалізація ITokenStream для ефективної роботи з токенами, які повністю завантажені в оперативну пам'ять (у вигляді масиву).

Sampling/BatchWindowSampler.cs: Відповідає за математику випадкового вибору. Гарантує, що випадкові індекси початку вікон обираються коректно.

Опис Методів
Бібліотека концентрується навколо головного методу генерації даних:

GetBatch(int batchSize, int blockSize, Random? rng):
Основний метод для отримання даних. Приймає:

batchSize — кількість незалежних послідовностей у батчі.

blockSize — довжина контекстного вікна (максимальна кількість токенів для передбачення наступного).

rng — опційний генератор випадкових чисел.
Результат: Повертає сформований об'єкт типу Batch з готовими парами контексту та цілі.

Очікування інтеграції для Етапу 2
Командам з Етапу 2 під час використання нашої бібліотеки слід враховувати наступні правила та гарантії:

Взаємодія через абстракції: Отримання тренувальних даних має відбуватися виключно через інтерфейс IBatchProvider.

Детермінованість тренування: Бібліотека повністю підтримує відтворюваність результатів. Якщо в метод GetBatch передати екземпляр Random з фіксованим seed, згенеровані батчі будуть абсолютно ідентичними при кожному запуску.

Безпека меж (Valid Windows): Наша логіка вибірки (BatchWindowSampler) жорстко контролює індекси. Ми гарантуємо відсутність помилок виходу за межі масиву (Index Out Of Bounds) при читанні потоку.

Обробка коротких потоків (Edge Cases): Ми передбачили крайові випадки. Якщо вхідний потік токенів виявиться меншим за запитаний blockSize, бібліотека відпрацює цей сценарій коректно (без падіння застосунку).

Тестове покриття: Наш код повністю покритий Unit-тестами. Протестовано формування нормального батчу, обробку коротких потоків та детермінованість вибірки при фіксованому seed.
